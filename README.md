# 产地加工场景 Demo

这是一个水果打包加工厂的移动端 demo，用两页完成“原料投入 -> 分选打包 -> 库存变化 -> 成本结转”的最小闭环。

## 运行方式

```bash
npm install
npm run dev
```

启动后打开终端提示的本地地址即可访问。页面按手机版设计，桌面打开时会以手机宽度居中展示。

## 业务理解

一批原料果进入加工环节后，会被分选成多种规格的成品果，同时产生次果和损耗。这个过程同时影响库存和成本：

- 库存：原料果 A、B 被消耗，成品果和次果增加，损耗只记录不入库。
- 成本：原料果 A、B 的采购成本需要结转到加工结果中，不能只展示重量变化。

本 demo 把一次加工视为一个批次，批次内录入原料投入、成品箱数、次果重量、损耗重量，结果页统一展示库存变化和成本分摊。

## 库存处理方案

- 原料果 A：按录入斤数扣减库存。
- 原料果 B：按录入斤数扣减库存。
- 成品果：按 `规格斤数 × 箱数` 计算入库重量，支持 5 斤装、8 斤装、10 斤装。
- 次果：作为低等级可销售或可处理产出，按斤数入库。
- 损耗：没有库存价值，不作为库存增加，但保留重量和损耗率。

系统会校验总产出重量是否超过总投入重量。若超过，则禁止查看结果；若小于投入，则在结果页展示未分配重量，提示批次数据未完全平衡。

## 成本处理方案

已知原料成本：

- 原料果 A：2 元/斤
- 原料果 B：3 元/斤

计算逻辑：

```text
原料总成本 = 原料果A重量 × 2 + 原料果B重量 × 3
加权单位成本 = 原料总成本 ÷ 总投入重量
结果分摊成本 = 对应结果重量 × 加权单位成本
```

成品果、次果、损耗都按重量分摊原料成本。其中成品果和次果进入库存成本，损耗单独展示为加工损耗成本，不进入库存。

## 主动补充的规则和假设

- 不接入数据库，数据只在前端内存中流转。
- 不考虑人工、包材、水电等加工费用，只处理原料成本。
- 原料 A/B 混合加工后不追踪具体进入哪个规格，统一按加权平均成本结转。
- 次果仍有可处理价值，因此作为库存产出参与成本分摊。
- 损耗没有库存价值，但需要归集成本，用于反映加工效率和损耗影响。
- 成品规格固定为 5 斤装、8 斤装、10 斤装，不做动态规格维护。
- demo 默认提供一组示例数据，打开后即可演示，也可以手动修改。
