# 产地加工场景 Demo

一个面向水果打包加工场景的移动端 demo，用于演示一批原料果加工后，库存数量和成本如何从原料结转到成品、次果和损耗。

- 线上预览：https://xiaoli-lhx.github.io/origin-processing-demo/
- 技术栈：Vite + React + TypeScript
- 页面形态：手机版两页，录入页 + 结果页
- 数据方式：前端模拟数据，不接入数据库

## 一、业务场景理解

在产地加工场景中，一批原料果进入加工厂后，会经过分选和打包，最终形成多种结果：

- 成品果：例如 5 斤装、8 斤装、10 斤装，可以作为标准商品入库。
- 次果：品质或规格不满足标准成品要求，但仍可能低价销售、二次加工或单独处理。
- 损耗：加工过程中不可销售、不可入库的部分，例如坏果、破损、杂质、称重差异等。

因此，这个场景不是简单的表单录入，而是一个加工批次中的库存和成本结转问题。

我将一次加工理解为一个独立批次：

```text
原料投入 -> 加工分选 -> 成品果 + 次果 + 损耗
```

系统需要回答两个核心问题：

```text
1. 库存发生了什么变化？
2. 原料成本最终结转到了哪里？
```

## 二、库存处理方案

本 demo 中，库存按加工结果进行处理：

| 类型 | 库存处理 | 说明 |
| --- | --- | --- |
| 原料果 A | 库存减少 | 加工消耗 |
| 原料果 B | 库存减少 | 加工消耗 |
| 成品果 | 库存增加 | 按规格箱数折算重量后入库 |
| 次果 | 库存增加 | 作为低等级产出入库 |
| 损耗 | 不入库 | 只记录重量、损耗率和损耗成本 |

录入页会校验投入和产出之间的重量关系：

```text
总投入重量 = 原料果 A 重量 + 原料果 B 重量
总产出重量 = 成品果重量 + 次果重量 + 损耗重量
```

处理规则：

- 如果总产出重量大于总投入重量，说明数据不合理，禁止查看结果。
- 如果总产出重量小于总投入重量，说明存在未分配重量，页面会提示复核。
- 未分配重量可以一键计入损耗，使批次重量达到平衡。

## 三、成本处理方案

题目给定：

```text
原料果 A 成本价：2 元/斤
原料果 B 成本价：3 元/斤
```

本 demo 暂不考虑人工、包材、水电、设备折旧等加工费用，只处理原料成本。

计算逻辑如下：

```text
原料 A 成本 = 原料 A 重量 × 2
原料 B 成本 = 原料 B 重量 × 3
原料总成本 = 原料 A 成本 + 原料 B 成本
加权单位成本 = 原料总成本 ÷ 总投入重量
```

加工完成后，原料成本按重量分摊到成品果、次果和损耗中：

```text
成品果成本 = 成品果重量 × 加权单位成本
次果成本 = 次果重量 × 加权单位成本
损耗成本 = 损耗重量 × 加权单位成本
```

其中：

- 成品果成本进入成品库存。
- 次果成本进入次果库存。
- 损耗成本不进入库存，但作为加工损耗成本单独展示。

这样可以形成成本闭环：

```text
原料总成本 = 成品果成本 + 次果成本 + 损耗成本
```

## 四、页面说明

### 录入页

录入页用于填写一个加工批次的基础数据：

- 加工批次号
- 加工日期
- 原料果 A 投入斤数
- 原料果 B 投入斤数
- 5 斤装箱数
- 8 斤装箱数
- 10 斤装箱数
- 次果斤数
- 损耗斤数

页面会实时计算投入重量、产出重量和原料成本，并提示批次是否平衡。

### 结果页

结果页用于展示加工批次的计算结果：

- 加权单位成本
- 投入成本
- 成品入库重量
- 次果入库重量
- 损耗率
- 原料扣减
- 成品和次果入库
- 损耗成本
- 各成品规格的分摊成本和单箱成本

## 五、主动补充的规则和假设

题目没有给出完整商业系统规则，因此 demo 中补充了以下假设：

1. 一次录入代表一个完整加工批次。
2. 原料果 A、B 混合加工后，不追踪具体流向到哪个规格。
3. 原料成本采用加权平均方式结转。
4. 成品规格固定为 5 斤装、8 斤装、10 斤装。
5. 次果仍有可处理价值，因此作为库存产出。
6. 损耗没有库存价值，因此不入库，但需要承担成本。
7. 不接入数据库，数据只在前端内存中流转。
8. 页面内置默认示例数据，便于直接演示。

其中，成本分摊方式、次果入库、损耗不入库但承担成本等规则，是基于业务场景做出的主动判断。
